# 《Web 程序设计》课程设计文档（项目策划书）

**主题： 中国古代建筑成就 ——《古建之光》Web 展示平台**

---

## 1 选题及作品名称

- **选题： 中国古代建筑成就的 Web 可视化展示网站**
- **作品名称： 《古建之光》（Glory of Ancient Architecture）**

---

## 2 要实现的需求

### 基础功能

1. 用户登录与注册模块  
2. 至少 2 个展示模块  
3. 内容从文件或数据库读取  

### 进阶功能（优秀要求）

1. 后台管理系统  
2. 完整 4 个展示模块  
3. 页面风格统一  

---

## 3 功能模块设计

### 3.1 流程设计

#### 用户注册流程

1. 进入注册页面  
2. 输入信息  
3. 校验格式  
4. 写入数据库  
5. 注册成功  

#### 用户登录流程

1. 输入账号密码  
2. 后端验证  
3. 登录成功（管理员跳后台）  

#### 用户浏览流程

1. 进入模块  
2. 加载内容  
3. 展示图文介绍  

#### 管理员 CRUD 流程

1. 登录后台  
2. 选择模块  
3. 添加/编辑/删除内容  

---

### 3.2 界面设计

包含以下页面：

- 首页  
- 登录注册页  
- 建筑分类列表页  
- 建筑详情页  
- 后台管理页面  

统一中式配色：古典红 + 金色 + 灰白。

---

### 3.3 功能设计

#### 用户认证

- 登录  
- 注册  
- 权限判断  

#### 内容展示模块（四类）

1. 皇宫  
2. 桥梁  
3. 官府  
4. 民居  

#### 后台管理模块

- 内容增删改查  
- 图片上传  

---

```
用户模块

```

---

### 3.4 数据库设计

#### users（用户表）

| 字段     | 类型    | 长度 | 含义            | 主键 |
| -------- | ------- | ---- | --------------- | ---- |
| id       | int     | 11   | 用户 ID         | PK   |
| username | varchar | 20   | 用户名          |      |
| password | varchar | 50   | 密码            |      |
| email    | varchar | 60   | 邮箱            |      |
| role     | tinyint | 1    | 0=普通 1=管理员 |      |

#### buildings（建筑内容表）

| 字段        | 类型     | 长度 | 含义     | 主键 |
| ----------- | -------- | ---- | -------- | ---- |
| id          | int      | 11   | 内容 ID  | PK   |
| category    | varchar  | 20   | 分类     |      |
| title       | varchar  | 100  | 标题     |      |
| description | text     | —    | 描述     |      |
| img_url     | varchar  | 255  | 图片地址 |      |
| created_at  | datetime | —    | 创建时间 |      |

---

### 3.5 接口设计

#### 登录接口

- URL: `/api/login`
- 方法: POST

#### 注册接口

- URL: `/api/register`
- 方法: POST

#### 获取建筑内容

- URL: `/api/buildings?category=xx`
- 方法: GET

#### 后台新增内容

- URL: `/api/admin/addBuilding`
- 方法: POST
